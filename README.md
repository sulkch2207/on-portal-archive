# on-portal-archive

本仓库用于归档和发布多个独立的 HTML 页面。这些页面以静态方式存储，可通过 GitHub Pages 或其他静态托管服务直接访问。

## 项目简介

这是一个通用型的页面归档仓库，主要用于保存和展示独立制作的 HTML 文件。仓库不针对特定网站或域名，仅作为内容的集中存放与发布场所。

## 目录说明

```
/
├── pages/          # 存放归档的 HTML 页面文件
├── assets/         # 页面引用的静态资源（样式、脚本、图片等）
├── index.html      # 仓库根目录入口页面（可选）
└── README.md       # 本说明文件
```

- `pages/` 目录下每个 HTML 文件代表一个独立归档页面。
- `assets/` 目录按需组织，用于存放页面所需的公共或私有资源。
- 根目录下的 `index.html` 可作为导航页或默认入口，可根据需要添加或删除。

## 页面归档说明

- 所有页面均为独立的 HTML 文件，不依赖后端服务。
- 页面内容应保持完整，包括必要的样式与脚本。
- 归档页面建议使用相对路径引用资源，以确保在不同环境下均可正常访问。
- 如需新增归档页面，请将文件放入 `pages/` 目录，并检查资源引用路径是否正确。

## 维护说明

- 本仓库由维护者自行管理，不接受外部提交请求。
- 页面内容更新时，请确保保留历史版本（如有需要）。
- 仓库结构保持简洁，避免引入无关文件或依赖。
- 如需启用 GitHub Pages，请在仓库设置中配置为从 `main` 分支发布。

## 许可

本仓库内容仅供归档与展示使用，不构成任何形式的授权或声明。具体页面的使用条款请参考对应文件中的说明。
